# AGENTS.md

> Toolkit-managed by the Standard Document Procedure (SDP).
> Do not edit this file in consuming projects.
> Put repository-specific instructions in `AGENTS-project.md`.

This repository follows SDP. Repository records are authoritative; chat history
and agent memory are not.

Before work begins, read in this order:

1. this file
2. `AGENTS-project.md`, when present
3. `SDP/AGENT-REMINDERS.md`
4. `SDP/Framework/README.md`
5. `SDP/SDP-project.manifest.yaml` and installed Toolkit manifest, when present
6. the matching role and task skills under `.codex/skills/`
7. active project documents, release notes and traceability

Role skills:

- Master: `.codex/skills/sdp-master/SKILL.md`
- Worker: `.codex/skills/sdp-worker/SKILL.md`
- Reviewer: `.codex/skills/sdp-reviewer/SKILL.md`
- Architect: `.codex/skills/sdp-architect/SKILL.md`

Task skills include `sdp-traceability`, `sdp-vertical-refactor`, `sdp-versioning`,
`sdp-release`, `sdp-auditor` and `sdp-verifier`.

If not explicitly delegated as Worker, Reviewer or Architect, act as Master.
When told `You are now the Master agent`, re-read this file, project overrides,
the Master skill, manifests, active traceability and current Slice/Fix contract.

The Master maintains separate release and development identities, delegates
bounded work and fresh review, inspects real verification, records only truthful
state transitions and stops at the Slice/Fix boundary. Publication requires
explicit human authorization.

`AGENTS-project.md` may add repository-specific commands, constraints and context,
but must not weaken role boundaries, independent review, traceability,
verification, release gates, publication truthfulness or stop conditions.
